# Бот для управления расписанием трибун в Discord
## Описание
Бот для управления расписанием трибун в Discord - это специализированное решение для организации и координации трибун (стримов или прямых эфиров) на Discord-сервере. Он позволяет автоматизировать процесс создания расписания, регистрации участников и управления сессиями.
## Основные возможности
- 📅 Создание расписания трибун на две разные схемы недели:
    - Неделя 1: Понедельник, Среда, Пятница, Воскресенье
    - Неделя 2: Вторник, Четверг, Суббота

- 👥 Удобная регистрация участников через интерактивные кнопки
- 🔄 Автоматическое обновление сообщения с расписанием
- 🕒 Разделение трибун на временные части (21:00-22:00, 22:00-23:00, 23:00-00:00)
- 🔐 Автоматическое управление сессиями пользователей с таймаутами
- 🧩 Различные роли участников (ведущий, замена)
- 🛡️ Обработка ошибок и защита от некорректных действий

## Технические характеристики
- **Язык программирования**: JavaScript (Node.js)
- **Основные библиотеки**:
    - discord.js (v14.19.3) - взаимодействие с Discord API
    - node-schedule (v2.1.1) - планирование задач
    - nodemon (v3.0.1) - автоматическая перезагрузка при разработке

- **Формат хранения данных**: JSON
- **Файловая структура хранения**:
    - data.json - данные расписания
    - storage.json - служебные данные (ID сообщений)
    - config.json - конфигурация бота

## Установка и настройка
### Требования
- Node.js 16.9.0 или выше
- npm или yarn
- Discord аккаунт и созданное приложение в [Discord Developer Portal](https://discord.com/developers/applications)

### Шаги по установке
1. Клонируйте репозиторий:
   git clone https://github.com/username/tribunebot.git
   cd tribunebot
2. Установите зависимости
   npm install
3. Создайте файл конфигурации со следующим содержимым: `config.json`
   {
     "token": "ВАШ_ТОКЕН_БОТА",
     "channelId": "ID_КАНАЛА_ДЛЯ_РАСПИСАНИЯ"
   }
4. Включите привилегированные намерения для бота в [Discord Developer Portal](https://discord.com/developers/applications):
    - Перейдите в раздел Bot
    - Включите опции "SERVER MEMBERS INTENT" и "MESSAGE CONTENT INTENT"
5. Зарегистрируйте команды бота (требуется выполнить один раз):
   node deploy-commands.js
6. Запустите бота:
   npm start

## Использование
### Команды бота
- `/week1` - Создать расписание трибун на неделю 1 (Пн, Ср, Пт, Вс)
- `/week2` - Создать расписание трибун на неделю 2 (Вт, Чт, Сб)

### Процесс работы с ботом
1. Администратор запускает команду `/week1` или `/week2` для создания расписания
2. Бот публикует сообщение с расписанием и кнопками для регистрации
3. Участники нажимают на кнопки и выбирают:
    - День трибуны
    - Часть трибуны (1, 2 или 3 - запасная)
    - Роль (ведущий или замена)

4. После регистрации расписание автоматически обновляется
5. Администратор может отслеживать заполнение расписания

## Структура проекта
- - Основной файл бота `index.js`
- - Скрипт для регистрации slash-команд `deploy-commands.js`
- - Файл конфигурации (токен бота, ID канала) `config.json`
- - Хранение данных расписания `data.json`
- - Хранение ID сообщений `storage.json`
- - Информация о зависимостях проекта `package.json`

## Советы по разработке и расширению
- Для добавления новых команд используйте шаблон в `deploy-commands.js`
- Для изменения формата расписания редактируйте функцию `formatTribune`
- Для добавления дополнительных ролей измените массивы в соответствующих функциях
- Используйте для разработки (`npm run dev`) `nodemon`

## Решение проблем
- **Ошибка "Used disallowed intents"**: Необходимо включить привилегированные намерения в Discord Developer Portal
- **Бот не отвечает на команды**: Проверьте, зарегистрированы ли slash-команды через `deploy-commands.js`
- **Ошибки при обновлении сообщения**: Убедитесь, что у бота есть права на редактирование сообщений в канале
